# Step 2：确定文案类型

## 5 种文案类型

| 类型 | 定位 | 适用场景 |
|------|------|----------|
| **分享型** | 故事化案件分享，悬念开场、情感共鸣 | 短视频引流、品牌建设 |
| **分析型** | 深度案件分析，逻辑拆解、争议焦点 | 专业普法、树立权威 |
| **吃瓜型** | 奇葩/娱乐化案件，吃瓜式开场、层层反转 | 引流爆款、评论区互动 |
| **观点型** | 法律观点输出，反常识开场、观点先行 | 树立权威、干货引流 |
| **观点干货版** | 误区破除 + 本地实务 + 算账避坑，纯干货无案情 | 长期信任建设、IP护城河 |

## 选择流程

1. 确认用户有无指定类型 → 如有则使用
2. 如无 → 参考 01-input-processing 的自动推荐
3. 用户确认后 → 进入 Step 3 钩子路由
