# Step 3：撰写口播文案

根据选定的文案类型和钩子，按对应模板撰写。

## 3.1 分享型 — 故事化案件分享

**定位：** 悬念开场、情感共鸣，适合短视频引流。
**结构：** '他来找我的时候' → 事实 → 我们的判断 → 做了什么 → 结果 → 对观众说

| 段落 | 内容 | 视角 |
|------|------|------|
| 立场开场 | 钩子句 + 律师立场表态 | 一句话说清价值观 |
| 案件概览 | "当事人来找我的时候" + 当事人状态 + 发生了什么 | 当事人第一视角 |
| 法律分析 | "我们一看就知道" + 核心法律判断 | "我们"视角 |
| 行动过程 | 律师采取了什么行动 + 关键节点 | 行动线 |
| 结果宣告 | 法院怎么说 + 当事人现状 | 事实陈述 |
| 对观众说 | 如果遇到类似情况 + 互动引导 | 第二人称 |

**核心心法：**
- 开庭对抗快节奏，像在听一场紧张的对峙
- 拿出关键证据时说"我们当时拿出了XX"，制造翻转感
- 不说"律师说"，统一用"我们/我"

## 3.2 分析型 — 深度案件分析

**定位：** 逻辑拆解、争议焦点，适合专业普法。
**结构：** 反常识钩子 → 规则破题 → 案子佐证 → 法律干货

| 段落 | 内容 | 视角 |
|------|------|------|
| 反常识钩子 | "很多人都以为XX，但法律上不是这样" | 观众视角 |
| 规则破题 | "处理这种案件，核心就是看一个点" | 专业视角 |
| 案子佐证 | 简说一个案子：怎么发生的、法院怎么看 | 第三方 |
| 法律干货 | 总结+建议+互动 | 第二人称 |

**核心心法：**
- 规则必须具体：不说"法律有规定"，说"民法典第XX条"
- 案子佐证一笔带过，不超过 2 句话
- 法律干货要点列清楚，每条不超过 15 字

## 3.3 吃瓜型 — 奇葩/娱乐化案件

**定位：** 吃瓜式开场、层层反转，适合引流爆款。
**结构：** 吃瓜钩子 → 事实反转 → 法律翻转 → 感叹收尾

| 段落 | 内容 | 视角 |
|------|------|------|
| 吃瓜钩子 | "这个案子我看了都摇头" | 主观视角 |
| 事实反转 | 发生了什么 + 当事人怎么想 | 事实 |
| 法律翻转 | "但法院说——" + 法律依据 | 专业视角 |
| 感叹收尾 | 简短感受 + 互动 | 主观视角 |

**核心心法：**
- 反转感要强，逻辑链条清晰
- 不说教、不普法
- 法院的判决是最后的"反转点"
- 人设状态：我看了都摇头/我看了都叹气/我看了都想笑

## 3.4 观点型 — 法律观点输出

**定位：** 反常识开场、观点先行，适合树立权威。
**结构：** 反常识钩子 → 观点先行 → 案子佐证 → 延伸建议

| 段落 | 内容 | 视角 |
|------|------|------|
| 反常识钩子 | "XX情况下，法院不支持你" | 第一人称 |
| 观点先行 | "记住这句话——" + 核心观点 | 律师身份 |
| 案子佐证 | 简说一个案子：怎么发生的、法院怎么看 | 第三方 |
| 延伸建议 | "所以记住了" + 互动 | 第二人称 |

**核心心法：**
- 观点句前置且独立成段，观点句用划线强调
- 案子佐证不超过 2 句话
- 延伸建议要具体，不写"有法律问题找律师"

### 全类型通用心法（4.1 — 4.6）

#### 4.1 一致性检查
- 开头立场与结尾立场一致，不能"开头骂对面、结尾说公平"
- 全文第一人称统一用"我们/我"

#### 4.2 "用你说吗"原则
- 去掉"法律是公正的"、"法院是讲道理的"等废话
- 去掉"有法律问题找律师"等营销废话

#### 4.3 顺滑结构
- 段落之间必须有过渡句，不能跳跃
- 每段不能超过 3 句话

#### 4.4 句间表现标记
- 句尾标记：`[抬头看镜头]` `[放慢]` `[停顿2秒]` `[比划]`
- 标记位置：情感转折点、关键观点输出前、悬念揭示前
- 标记密度：全篇 3-5 处，均匀分布

#### 4.5 平台适配
在输出时根据发布平台调整：

| 要素 | 抖音 | 视频号 | 小红书 |
|------|------|--------|--------|
| 标题风格 | 冲击力钩子 | 悬念式 | 评论区风 |
| 字幕要求 | 大字+强对比 | 适中 | 花字+表情 |
| 时长偏好 | 15-30秒 | 30-60秒 | 30-60秒 |

#### 4.6 结尾设计
- **一定要给观众下一个看的理由**，不写"关注我，每天讲实用婚姻财产法律"
- **互动话术必须具体**，不给空泛的"点赞关注"

## 3.5 观点干货版 — "纯干货，无案情"

**与观点型的核心区别：**

| 维度 | 观点型 | 观点干货版 |
|------|--------|-----------|
| 驱动力 | 一个反常识观点 + 案子佐证 | 大众认知误区 -> 实务真相 |
| 结构 | 固定4段 | 无固定结构，6心法自由组合 |
| 案子 | 必须有，一笔带过 | 不需要，纯知识 |
| 人设 | 规则的翻译者 | 客观理性的避坑向导 |
| 适用 | 有典型案件时 | 任何法律题材均可 |

**通用核心创作逻辑（所有爆款统一内核）：**
> 打破大众情绪认知 -> 落地真实实务规则 -> 量化用户利弊得失 -> 塑造真诚专业人设

不迎合大众的主观情绪与固有误区，专门输出普通人搜不到、看不懂、被网络谣言误导的司法实务真相。

**六大创作心法（无模板，纯思维）：**

| 心法 | 核心 | 底层逻辑 |
|------|------|---------|
| ① 开篇心法 | 抓全民误区做认知反差 | 大众怎么以为，我就反向说真相 |
| ② 代入心法 | 抓用户高频行为做场景共情 | 先接纳焦虑，不批判，再纠正 |
| ③ 专业心法 | 抓权威依据破除主观抬杠 | 用官方口径堵质疑，不靠个人经验 |
| ④ 落地心法 | 抓地域实操打造差异化 | 通用法条人人可查，本地裁判才是稀缺 |
| ⑤ 价值心法 | 抓投入产出做避坑干货 | 教用户别瞎亏，不是教怎么赢 |
| ⑥ 人设心法 | 抓反营销感建长期信任 | 极致利他、真诚避坑是最高级引流 |

**爆款自查6问（写完后逐条核对）：**
1. 有冲突？——推翻了大众普遍相信的法律误区
2. 有画面？——还原了普通人真实的维权误区场景
3. 有权威？——核心观点有官方司法依据支撑
4. 有落地？——结合本地实操，区别于全网通用废话
5. 有价值？——帮用户省钱、省时间、避开维权大坑
6. 有人设？——客观中立，没有刻意营销收割用户

**终极一句话公式：**
> 用大众最深信的误区做流量入口 → 用官方+本地实务做专业信任 → 用算账避坑做干货价值 → 用利他真诚做账号人设
