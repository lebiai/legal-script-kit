# Step 1：案件识别与自动推荐

接收案件材料后，按以下顺序做自动推荐（可一键采用，无需逐条询问）。

## 1.1 案件类型自动识别

从材料内容推断，主要类型：

| 类型 | 特征 |
|------|------|
| 交通事故/人身伤害 | 涉及事故认定书、医疗记录、伤残鉴定 |
| 婚姻家事/离婚 | 涉及离婚协议、财产分割、子女抚养 |
| 刑事辩护 | 涉及起诉书、取保候审、量刑建议 |
| 合同/经济纠纷 | 涉及合同履行、违约金、欠款追讨 |
| 劳动工伤 | 涉及劳动合同、工伤认定、劳动仲裁 |
| 遗产继承 | 涉及遗嘱、法定继承、遗产分配 |
| 知识产权 | 涉及商标、专利、著作权侵权 |
| 房产纠纷 | 涉及房屋买卖、租赁、产权争议 |

## 1.2 文案类型自动推荐

| 案件类型 | 推荐文案类型 | 理由 |
|----------|-------------|------|
| 交通事故/人身伤害 | 分析型 / 观点型 | 适合讲赔偿计算和法律误区 |
| 婚姻家事/离婚 | 分享型 / 吃瓜型 | 有情感故事和反转空间 |
| 刑事辩护 | 观点型 / 分析型 | 适合输出反常识观点 |
| 合同/经济纠纷 | 分析型 / 观点干货版 | 适合算账避坑、讲实务规则 |
| 劳动工伤 | 观点型 / 观点干货版 | 打工人高频关注，适合误区破除 |
| 遗产继承 | 分享型 / 观点型 | 有戏剧性和反常识点 |
| 知识产权 | 分析型 / 观点干货版 | 专业门槛高，适合干货输出 |
| 房产纠纷 | 观点型 / 观点干货版 | 金额大、误区多，适合避坑 |

## 1.3 人设风格自动推荐

| 案件特征 | 推荐人设 | 理由 |
|----------|---------|------|
| 奇葩、反转、维权成功 | 犀利分析型 | 适合毒舌点评，观众爱看 |
| 婚姻家事、劳动纠纷 | 亲切邻家型 | 温和共情，观众有代入感 |
| 娱乐化案件 | 幽默吐槽型 | 轻松氛围，利于传播 |

## 输出

向用户确认自动推荐结果（一句话），用户可一键采用或调整。
